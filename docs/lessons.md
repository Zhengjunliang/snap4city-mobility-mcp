L33 **car 路线附带"终点附近空停车位": 远程 `service_search_near_gps_position` near dest, 仅 car, 并发进 routing 的 flat gather; 实时/类别/距离/pin 走真机 probe 校准, 缺则退化** —— 用户要开车时看终点附近空停车场。**工具**: referente 远程已有 `service_search_near_gps_position` (坐标+`categories`+`maxdistance`+`values=` 一次取每条 value), 加进 [EXPOSED_TOOLS](../src/snap4city_mobility_mcp/mcp_tools.py) 即经 `exec_tool` 透传; **不加** `get_service_categories` (只 probe 用裸 client, 运行期不调, 避死代码)。**触发**: [execute](../src/snap4city_mobility_mcp/orchestrator.py) `"car" in modes` 才搜 (显式 car 或未指定多模式)。**并发坑 (关键)**: 必须 **flat** `asyncio.gather(*route_coros, parking_coro)` 把停车放最后一个, **不能嵌套** `gather(gather(routes), parking)` —— FakeClient 是单队列 FIFO 且 `call_tool` 无 await 挂起点, flat 下协程按创建序跑完再下一个 (geocode×2 → routing(s) → parking, 确定性); 嵌套则外层 gather 先调度 parking task, 它会在 route 子任务前抢先 pop 队列 → 顺序乱、既有 order 断言测试红。routing 条目仍按 modes 序 extend, 停车条目 append 在所有 routing 之后 (`_extract_data` 只看 routing 名故不受扰)。**数据**: [_extract_parking](../src/snap4city_mobility_mcp/orchestrator.py) + [parse_parking_features](../src/snap4city_mobility_mcp/mcp_tools.py) 健壮读多 envelope (`features`/`Service.features`)、多空位 key; **距离客户端 Haversine 从 dest 算** (envelope 距离字段单位未验证, 不信); 排序: 有空位数者优先 (多者前) 再近, 无空位数 (实时未加载, 退化) 排后按近。`data.parking` 注入**独立于 `data.routes`** (car 在 ZTL 路线空 L8 时仍出停车, "开不进去但附近能停")。`data.parking` 同 [[L32]] origin/destination 是给本项目自己前端的字段非 referente 自创 (规则 #8)。**前端**: [dashboard](../frontend/mobility_advisor_dashboard.html) `renderParking` 聊天列出 (名+距+空位/`post. n/d`) + `drawParkingPins` 用 widgetMap `addSelectorPin` (读真源 `widgetsCommonFunctions.js`: `for n: trigger({type:"addSelectorPin",passedData:{query:superservicemap+"?serviceUri="+URI,queryType:"Default",desc,color1,color2}})` —— 逐条发是 Snap4City 官方模式, query 用具体 serviceUri 故单服务 pin 非整类 flood)。**L30 纪律未破**: pin payload/SuperServiceMap base/清 pin 事件 (`removeSelectorPin` 猜测, 真源无) 属未验证 → 全 try/catch, **聊天列表无条件渲染不依赖 pin**, 真机抓 PA.php 工作 payload 后校准。**真机 probe 结论 (`scripts/probe_parking.py`, 2026-06-26)**: (a) `PARKING_CATEGORY="Car_park"` **确认** (返城内 car park, `serviceType="TransferServiceAndRenting_Car_park"`, `tipo="Car_park"`); (b) **真 envelope** = `{"result": [[uris...], {"Services": {"features": [{geometry, properties:{name, serviceUri, distance:"0.125"(km 字符串), tipo}}]}}]}` —— features 嵌在 `result[1].Services.features` (复数 Services!), parser 用 `_find_feature_list` 递归找 (直接 features / `Service`|`Services` / `result` list/dict 都吃) 否则读不到; (c) **实时空位确实有 (但要二段取)**: search 结果**不**带空位; 真值在 orion/IoT car-park 实体 (`iot/orionUNIFI/...` 及部分 km4city resource)。第一次 probe 误判"无实时"是因为只测了第一个 URI (普通 POI 车库 `34cbe...` GARAGE VERDI 确实无实时)。`service_info` 的 `realtimeAttributes` 只是**字段 schema (元数据)**, 当前值要调 **`service_info_dev(serviceuri, fromtime="1-hour")`** → `realtime.results.bindings`(**最新在前**) `[0].freeParkingLots.value`(字符串)+`capacity.value` (实测 CarParkBeccaria 31/202)。**实现**: execute 对 search 出的近 N 个 (`_extract_parking` parse+Haversine 距离+排序+截 `PARKING_MAX`) **并发**调 `service_info_dev` 填 `free_spaces`/`total_spaces` ([_enrich_parking](../src/snap4city_mobility_mcp/orchestrator.py)), enrich 后**重排** (有空位者优先、多者前, 余按距离); 普通 POI 车库无 realtime → free 留 None (退化显位置)。enrich 调用**不进 audit** (内部, 非 LLM-facing)。最终 parking 经 `AdvisorState.parking` 传到 respond 注入 `data.parking` (前端 `renderParking` 列表展示, 含 `free/total liberi`)。**LLM 不喂停车** (`_results_view` 跳过 service_search 条目, RESPOND_SYSTEM 明令"别在文字提停车"): 前端列表已完整显示, LLM 再列就重复 (用户真机反馈"和下面重合") → 文字只答行程。**widgetMap 每个 addSelectorPin 弹"X ADDED TO MAP"蓝色反馈框 (parent DOM, widget 自带, commonFunctions 无抑制 flag) → 用户拍板当特性接受, 不处理。**距离客户端 Haversine 算 (envelope 距离字符串单位虽是 km 但仍自算避歧义)。`service_info_dev` 加进 [EXPOSED_TOOLS](../src/snap4city_mobility_mcp/mcp_tools.py); realtime 解析 = [read_parking_realtime](../src/snap4city_mobility_mcp/mcp_tools.py)。教训补充: "返空"别只测一个样本 — 同类实体里 POI 无实时、IoT 实体有, 要按 URI 类型分别测 (probe P3 先测 iot/orion); 且实时当前值常在 `service_info_dev` 的时序 bindings (newest-first) 里, 不在 `service_info` 的 attribute schema 里。教训: 加"附近找服务"类特性, 工具多半远程已有 (查 probe-native-tools.json 25 工具); 真机未知 (类别名/实时/envelope/widget pin) 一律 probe 先于解析 (parser/marker 后写) + 健壮多形状 + 优雅退化, 别让单形状假设或裸猜 widget 字段进线上。
